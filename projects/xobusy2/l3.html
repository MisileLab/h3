Python

인터프리터 방식 프로그래밍 언어

1. 특징

  1. 작성한 코드를 읽기 쉬움
  2. 중괄호를 사용하는 대신 Space 4문자 사용
  3. Type hinting이 없고 모든 것이 객체를 취급된다
  4. 반복을 지원하는 객체가 많고 만들기 쉽다

2. 장점

  1. 빠른 개발 속도
    2. C언어로 2년동안 완성 못한 프로젝트를 Python으로 한 달만에 해냈다는 이야기도 있을 정도로 개발 속도가 빨라진다.
    인터프리터 언어이면서 다양한 모듈, 자료형을 제공하기 때문이다.
    에러도 친절하다.
    C처럼 Segmentation Fault만 내주는게 아닌 어디서 어떻게 에러가 발생했는지 자세하게 설명해준다.
  2. 피드백의 용이성
  3. 과학 및 공화 친화성
    4. 파이썬은 과학과 공학 분야에서 필요한 여러 기능을 기본적으로 제공한다. 예를 들면
    1. 허수
    2. 64비트를 넘어가는 매우 큰 정수
    3. 정밀한 소수점과 유리수
    4. 행렬곱

3. 단점

  1. 느린 실행 속도
    2. Python은 C보다 느리다.
    사실 여기까지는 다른 인터프리터 언어도 똑같지만 심하게 느리다.
    얼마나 느리냐면, C보다 71.90배의 시간, 2.80배의 메모리를 더 소모한다.
그러니까 Rust 쓰세요
