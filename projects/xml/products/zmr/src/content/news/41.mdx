이 프로젝트는 Git Submodules의 대안으로서 다양한 프로그래밍 언어의 외부 의존성을 더 간단하게 관리할 수 있는 도구입니다.\
주요 사용 사례로는 공동 CI 자원 접근, 백엔드 애플리케이션과의 프론트엔드 코드 통합, 프로토콜 버퍼 가져오기가 포함됩니다.

## 주요 기능:
- **설치**: 바이너리 다운로드나 Go 툴체인을 통해 설치 가능.
- **사용법**: YAML 또는 JSON 파일로 의존성을 정의하고 `vdm sync` 명령을 사용하여 의존성을 동기화.
- **디렉토리 구조**: 동기화 후 의존성이 지정된 디렉토리 구조 내에 정리됨.
- **의존성**: 독립 실행형 바이너리로 작동하며, git 기반 의존성에는 Git 필요.
- **인증**: 인증 처리는 사용자에게 맡김.

## 향후 개선 사항:
- 원격 변경 관리 방식을 개선.
- `.git` 디렉토리를 기본으로 유지하는 옵션 추가.
- 지원하는 원격 유형을 `git`과 `file` 외에 확장 예정.
